What is Landlab?

Landlab is a Python-based modeling environment that allows scientists and students to build numerical landscape models. Designed for disciplines that quantify earth surface dynamics such as geomorphology, hydrology, glaciology, and stratigraphy, it can also be used in related fields.

Landlab provides components to compute flows (such as water, sediment, glacial ice, volcanic material, or landslide debris) across a gridded terrain. With its robust, reusable components, Landlab allows scientists to quickly build landscape model experiments and compute mass balance across scales.

Landlab is described in more detail by Hobley et al. in the 2017 paper Creative Computing with Landlab.

Examples

The following models can easily be built with Landlab's current capabilities:

  • A landscape evolution model using linear diffusion and the stream-power model
  • A model that explores the flexural response to the growth and recession of glaciers
  • An ecohydrology model in which vegetation on two sides of a valley grows and dies in response to stochastic storms and solar forcing throughout the year
  • A model that routes hydrographs across a watershed based on rainfall inputs

    • Installing Landlab

    To install Landlab, you will need a Python distribution that includes

  • Python 2.6+ or 3.5+ (NB: support for Python 2 will be dropped in Landlab v.2)
  • NumPy 1.9 or greater
  • SciPy 0.14 or greater
  • netCDF4 (will run without, but recommended)
  • Cython (only required if building Landlab from source code)
  • an up-to-date Python package manager (such as conda or PyPI, aka "pip")
  • a compiler that Python can recognise (through the Xcode app on a mac)

    • To fulfill these requirements we recommend a preassembled scientific Python distribution such as Continuum IO’s Anaconda. We recommend choosing Python 3 over Python 2 when offered the choice. As well as the Python package and associated libraries, the Anaconda installer also puts a suite of useful Python tools on your machine, including a graphical interface for coding. However, what we are mostly after is the very powerful command line tool for managing your system Python called conda. This gets installed automatically, and we'll be exploiting it below. (Note that you can also install conda without all the associated tools - AKA "miniconda" - if you want, via other approaches.)

    Once your Python distribution is complete and up to date, install Landlab.

    Installation Options

    You need to choose between two installation options for Landlab at this stage:

  • Using Anaconda - installation of a pre-compiled Landlab release. It's quick and easy but you will not be able to access and edit the code itself.
  • Developer Installation - forking Landlab's github repository. This is the preferred option if you want to modify code and commit back to Landlab.

    • We anticipate most users will want to choose an Anaconda install, at least while becoming familiar with Landlab.

    Installing Landlab Using Anaconda

    We recommend running Landlab inside its own conda environment. This keeps it from interfering with any other Python software you may have on your machine.

    To do this, open a terminal (or, on a PC, the command prompt—make sure it is a true DOS "cmd" prompt, not, e.g., Cygwin) and type the following:

    > conda create -n landlab_env landlab -c conda-forge

    You can now activate this environment and use Landlab inside it. Do this with:

    > conda activate landlab_env

    And proceed to use Landlab. When you're done, close up your environment with:

    > conda deactivate

    There are also two install options we don't formally recommend. You can still install Landlab directly without an environment using conda install landlab -c conda-forge, and there is also an up-to-date pip distribution for users who would prefer that.

    More detailed instructions for installing Landlab with Anaconda's Python distribution are available in the reference manual.

    Developer Installation—Installing from Landlab Source Code

    For users who have gotten a feel for Landlab and want to keep up with the absolute latest Landlab developments and contribute code back to the Landlab repository, these directions are available as part of the Landlab guide for developers.

    Updating Landlab

    As Landlab is still relatively early in its development cycle, the code will update fairly frequently with new releases. To take advantage of new features and new library additions (as well as bug fixes, ahem), we recommend you update Landlab regularly.

    Whenever you update Landlab, we strongly recommend you also update your Python package! For Anaconda, use the conda package manager from a command prompt. Assuming you have Landlab installed in a conda environment, be sure to activate the environment first. [Remember, on a PC, do all this in a Command Prompt window or through Anaconda; do not use Cygwin.]

    > conda update --all

    Once you've done this, simply update landlab, once again specifically from the conda-forge conda channel:

    > conda update landlab -c conda-forge

    Older Installations of Landlab

    We used to recommend installing landlab outside an environment, using either the pip package manager or through a conda channel called -c landlab. Notably, if you installed through the conda landlab channel rather than conda-forge, you won't be able to access updates any more. If you're now using Landlab inside its own environment, having these older versions around won't matter too much. However, to minimise confusion, you may wish to remove them. Simply do this:

    > pip uninstall landlab

    and/or

    > conda uninstall landlab

    from outside your environment. The Landlab installation inside your environment should be unaffected.
